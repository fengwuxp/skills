# 资源炼技

## 定位

把异构材料转化为**可追溯、可执行、可验证、可归位**的能力候选。产物不是材料摘要，也不以创建新 Skill 为默认目标；先提炼能力单元，再为每个单元选择最小权威落点，允许零能力。

本能力只负责从资源到能力资产候选的提炼与验证，不接管材料所属领域的事实裁决，不替代最终内容创作、业务建模、工程实现或发布审批。

## 输入门禁

开始前确认：

- 可实际读取的材料正文、源码、规范或产物，而不是标题、二手摘要或模型记忆。
- 目标使用者、复用场景、目标环境、写入范围、验收方式和停止条件。
- 来源、版本或日期、许可、隐私和敏感信息边界；缺失时标为待确认，不自行补全。
- 用户是否只要求摘要、成文、领域建模或局部编辑；属于这些场景时不触发本能力。

## 核心流程

1. **定义能力问题**：写清要缩小什么重复错误、服务什么复用场景，以及什么证据才算完成。
2. **先做整体理解**：识别资源目标、结构、关键术语、局限和可能力化范围，避免从局部片段直接制造候选。
3. **建立来源矩阵**：实际读取材料，记录类型、来源锚点、版本、时效、许可和可用范围；区分事实、推断、冲突与待确认。
4. **提炼并筛选能力单元**：建立候选池，提取触发、输入、步骤、失败分支、停止条件、输出与验收；按证据支撑、外推能力和非平庸性决定接受、拒绝或继续补证据。
5. **归位并验证**：逐项归入新 Skill、增强既有 Skill、reference、script、asset、fixture 或任务内不沉淀；用硬负例、相邻能力混淆和真实产物检查触发与执行质量。

详细字段、候选生命周期、冲突处理、归位规则和验证方法读取 `references/distillation-contract.md`。需要校验结构化候选时直接运行 `scripts/check_capability_candidate.py --file <candidate.json>`。涉及本能力采用的外部方法来源及吸收边界时读取 `references/source-map.md`。

## 场景路由

| 场景 | 处理 |
| --- | --- |
| 教程、视频、代码、文档和产物共同描述一项能力 | 读取提炼契约，先做整体理解，再建来源矩阵、提炼和归位 |
| 单篇材料只需摘要、改写或正式成文 | 停止本流程，交给对应成文任务 |
| 材料用于建立具体业务域或工程域模型 | 停止本流程，交给领域事实与交付任务 |
| 已知只需修改某个既有资产 | 直接做局部编辑，不重新运行完整提炼流程 |
| 证据不足、许可不明、无法验证或没有稳定复用价值 | 输出待确认或不沉淀结论，允许零能力 |

## 最小输出

```text
提炼目标：
来源与证据边界：
能力单元：
- 名称 / 稳定职责：
- 触发 / 非触发：
- 输入 / 前置条件：
- 步骤 / 失败分支 / 停止条件：
- 输出 / 验收：
- 来源锚点 / 版本 / 许可：
- 建议落点：
冲突与待确认：
验证证据：
拒绝或不沉淀项：
```

输出规模跟随有效能力单元数量；没有合格单元时直接说明零能力，不用模板填充内容。

## 红线

1. 不把摘要、知识目录、观点集合或提示词堆叠冒充可执行能力。
2. 不复制来源正文、原图、作者口吻、密钥、客户数据、受限代码或许可不明内容。
3. 不只凭标题、搜索摘要、历史记录或工具总结写成已读取结论。
4. 不为每个材料创建新 Skill；已有权威位置能够承载时优先增强既有资产。
5. 不把来源冲突静默抹平；按版本、适用范围和证据强度保留差异，无法裁决时停止。
6. 不默认联网，不自行获得读取私有目录、安装、Git、同步、部署、删除或发布权限。
7. 候选不等于准入；不得自动安装、同步、提交或晋升。
