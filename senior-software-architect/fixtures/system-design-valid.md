# 订单回调系统分析设计

## 一、需求背景

背景：订单回调处理慢，问题是高峰期积压。不做的风险：生产延迟影响商户确认。

## 二、目标与边界

目标：降低 RT。非目标：不迁移数据库。系统边界、数据边界和安全边界保持清晰。

## 三、概要设计

概要设计：核心方案是拆分回调编排与状态回写。关键依赖包括订单服务、通知服务和消息队列；同步与异步关系明确。

## 四、详细设计

详细设计包含模块、类设计、接口设计和数据设计。

## 五、流程状态与非功能

状态机覆盖待处理、处理中、完成和失败。主流程、异常流程、补偿流程和人工介入路径齐全。非功能覆盖性能、容量和可用性。

## 六、测试与研发计划

测试设计包含单元测试、集成测试、契约测试和回归测试。研发计划包含负责人、里程碑和验收方式。
