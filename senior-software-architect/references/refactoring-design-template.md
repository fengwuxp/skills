# 重构设计模板

本文只用于跨模块、公共契约、数据迁移、核心链路替换、双轨切流或旧能力退役。局部、行为保持且可测试的重构使用实现任务卡和测试保护，不创建独立重构设计文档。

模板占位符统一使用 `〈...〉`；正式文档不得保留未替换占位符。

## 使用时机
- 需要兼容旧接口、消息、数据、客户端或运营工具，并通过双写、回填、影子读、灰度切流或 MIG 切片迁移。
- 目标结构已有系分依据，但从当前状态迁移到目标状态仍有独立风险。
## 不适用场景
- 单文件、单类或私有方法整理，且外部行为不变、现有测试可保护。
- 新增产品行为先更新产品设计和系分；局部取舍优先使用 ADR。
## 读取后必须产出
- 重构准入结论、当前证据、目标结构、行为不变量、MIG 切片、迁移规则、验证证据、回滚和旧能力下线条件。
## 需要继续读取的 reference
目标结构读 `system-analysis-template.md`，演进策略读 `evolutionary-architecture.md`，取舍读 `adr-and-tradeoff.md`，测试读 `testing.md`，生产保障读 `production-readiness.md`。
## 按任务读取索引
| 任务 | 优先读取 | 跳过 |
| --- | --- | --- |
| 判断是否独立设计 | 准入、当前证据、行为边界 | 不展开 MIG |
| 核心链路或数据迁移 | 全文 | 不复制目标系分和排期 |
| 局部行为保持重构 | 不使用本文 | 使用任务卡和测试 |
## 一页式正式模板

正文只保留当前有效迁移结论；探索、被拒方案和完整排期进入 ADR、Decision Log 或项目计划。复制后按真实风险裁剪：

```markdown
# 〈规范主题〉重构设计
- 状态 / owner / 评审 owner：〈Draft / Review / Approved；角色〉
- 产品设计 / 目标系分 / 源码测试运行证据：〈路径或锚点〉
- 事实 / 推断 / 待确认 / 范围外不做：〈分层说明〉
## 一、重构准入
外部行为变化及产品 AC：〈无则写行为保持〉；为什么局部修改或 ADR 不足：〈证据〉；影响业务入口、模块、契约和数据：〈范围〉；重构目标 / 非目标：〈边界〉。
## 二、当前问题与目标边界
当前结构、调用链、缺陷/事故/运行指标、承重行为和兼容对象：〈证据〉；目标结构、行为不变量、公共契约不变量、保留/替换/删除范围及复杂度停止条件：〈结论〉。
## 三、迁移规则
| 主写方/事实源 | 双写失败 | 回填/dry-run/审计 | 数据校验/对账 | 影子读 | 灰度切流/暂停 | 回滚 | 最长共存期/旧能力下线条件 |
| --- | --- | --- | --- | --- | --- | --- | --- |
|  |  |  |  |  |  |  |  |
时间边界：重启后记得什么；重放时如何避免重复副作用；状态不明时如何恢复、补偿、隔离或人工接管。

| MIG-001 | 〈可独立交付结果〉 |  |  |  | 〈测试/指标/对账〉 |  | 〈执行/验证〉 |
## 五、规则落地与影响
| 规则/不变量 | 当前承接 | 目标承接 | 迁移期保护 | 验证证据 |
| --- | --- | --- | --- | --- |
|  |  |  | 〈双轨守卫/开关〉 | 〈测试/指标/对账〉 |
## 六、验证与生产保障
特征测试 / 契约测试 / 单元集成回归 / 数据校验：〈证据〉；监控告警 / 灰度准入暂停退出阈值 / 回滚恢复 / 人工验收 / 残余风险：〈结论〉。
## 七、Engineering Handoff
本轮 MIG / 第一实施切片：〈目标〉；写入/只读范围和禁止事项：〈边界〉；行为不变量/公共契约：〈保护对象〉；第一批失败测试/验证命令：〈证据〉；执行 owner/验证 owner/停止条件：〈责任〉。
## 八、参考资料与证据索引（按需）
沿用系分模板同名索引的“类型、资料名称、链接或路径、版本/日期、用途与核验状态”；外部文档使用可读 Markdown 链接。引用标识为可选，仅在多次引用、迁移或验证追踪、资料混淆或审计要求时增加；已在正文清楚标注时省略本节，AI 推理、讨论草稿和被拒方案进入过程资产。
```

若行为变化没有产品来源，或目标结构没有系分来源，停止在待确认；MIG 必须可独立验证、暂停和回退。该交接不是 Execution Grant、Git 授权或上线批准。
