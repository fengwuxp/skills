# 经世决策实践

## 使用时机

当用户需要把经典判断镜片落到个人、组织或项目的现实决策、协作、成长与行动取舍时读取。本文细化六步工作法和经世决策卡。

## 不适用场景

- 已有明确动作且一步可以完成，无需重新做决策校准。
- 纯工程、产品、法律、医学、投资或合规专业判断，应先由对应专业能力负责。
- 用户只要古籍考据或文化表达，不需要现实行动。

## 读取后必须产出

- 一张经世决策卡，包含事实、名相、时势、取舍、行动、止损和验证。
- 采用 1-3 个镜片的理由，以及未采用其它镜片的跳过说明。
- 明确下一责任人、反馈时间或停止条件。

## 需要继续读取的 reference

- 选择经典镜片读 `classical-lenses.md`。
- 需要引用经典、处理高风险问题或发现证据冲突时读 `evidence-boundaries.md`。
- 需要核对来源边界时读 `source-map.md`。

## 按任务读取索引

| 任务 | 优先读取 | 跳过 |
| --- | --- | --- |
| 两难选择 | `察实`、`审时`、`权衡`、`行验` | 不先列满所有框架 |
| 组织协作 | `正名`、`权衡`、`行验` | 不设计操控话术 |
| 长期成长 | `察实`、`审时`、`化` | 不把一次状态写成性格宿命 |
| 项目取舍 | `正名`、`审时`、`行验` | 不替代产品或工程准出 |

## 察实

建立四栏事实边界：已知事实、合理推断、待确认项、范围外不做。优先读取用户材料、合同、源码、日志、数据或真实反馈；不能从“古人会怎么说”推导现实情况。

影响方向的关键事实缺失时，先问一个决策问题；不影响最小可逆行动时，可以标注假设后继续。

## 正名

依次确认：

1. 当前讨论的对象和问题究竟是什么。
2. 成功、失败、合作、完成、稳定等词分别以什么事实为准。
3. 谁拥有决定权、执行责任、验证责任和失败后果。
4. 口头承诺、制度名称和实际行为是否一致。

名相不清时停止扩展方案，先改成可观察、可负责和可验证的表达。

## 审时

识别三个时间维度：

- **不变约束**：价值底线、法律责任、安全边界和不可承受损失。
- **变化轴**：资源、关系、需求、能力、环境和反馈正在怎样变化。
- **行动窗口**：现在必须做什么、可以等待什么、何时重新判断。

把阶段写成当前事实，不把五行或五德名称当作时间证明。

## 权衡

从 `classical-lenses.md` 选择最少镜片，每个镜片只保留一个关键问题。对候选方案检查：

- 收益由什么事实支撑。
- 代价由谁承担，是否可逆。
- 最坏失败是什么，止损是否真实可执行。
- 反面证据和替代解释是什么。
- 是否存在更小、更快得到真实反馈的试验。

不追求“古今思想统一”，只追求现实决策更清楚。

## 行验

行动必须写清：动作、责任人、时间、写入或影响范围、反馈源、成功信号、失败信号、停止或回退。优先选择最小可逆动作；只有不可逆选择确有必要且人类责任人明确确认时才推进。

## 化

在约定反馈点回看：事实是否变化、假设是否被证伪、行动是否有效、是否触发停止。一次心得不沉淀为长期规则；重复出现并经验证的经验才进入决策记录、团队规则、知识库或个人实践。

## 经世决策卡

```text
事实与待确认：列事实、推断、关键缺口。
名相与责任：定义问题、成功标准和责任人。
时势与变化：列不变约束、变化轴和行动窗口。
关键取舍：写采用的镜片、收益、代价与反证。
最小行动：写动作、责任人、时间和影响范围。
止损与停止：写最坏失败、阈值、回退或交还条件。
验证与复盘：写反馈源、成功/失败信号和复盘时间。
```

## 常见失败

- **套框架**：先选名句再找现实例子。修正为先察实，再选镜片。
- **求圆融**：把冲突双方都说成有道理。修正为明确责任、损失和底线。
- **过度解释**：一口气展开全部经典。修正为最多三个镜片，每个一个问题。
- **只有道理**：没有动作和反馈。修正为补最小行动、止损与验证。
- **假装确定**：材料不足仍下结论。修正为标注待确认或停止。
