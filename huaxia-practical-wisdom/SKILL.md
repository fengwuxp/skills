# 华夏经世智慧

## 定位

以华夏经典智慧为判断镜片，帮助用户在真实事实、责任和风险约束下完成现实决策与行动校准。经典用于问问题、辨关系和审时度势，不能替代事实、专业知识、验证或人类责任。

本 Skill 是独立专业能力，不是人格、流程总入口或古典知识百科。`wise-agent` 可在跨阶段任务中按需装载它；单一现实决策也可直接使用。

## 输入输出契约

开始前确认：当前问题、已知事实、目标、约束、相关主体、时间范围、最坏损失和可验证反馈。材料不足时先从用户材料自答；真正影响取舍的未决项一次只问一个。

交付一张经世决策卡：

```text
事实与待确认：
名相与责任：
时势与变化：
关键取舍：
最小行动：
止损与停止：
验证与复盘：
```

简单问题可以压缩表达，但不得省略事实边界、行动和验证。

## 经世六步

1. **察实**：区分事实、推断、待确认和范围外不做，不从经典名句反推现实事实。
2. **正名**：确认对象、目标、角色、责任和完成之名是否对应实际。
3. **审时**：识别不变约束、变化轴、时位、阶段和可逆窗口。
4. **权衡**：只选 1-3 个能缩小错误空间的经典镜片，并检查反面、过度和最坏失败。
5. **行验**：给出最小可逆行动、止损线、反馈源、验证方法和下一责任人。
6. **化**：根据真实反馈修正判断；经反复验证的经验才进入规则、知识库或长期实践。

## 镜片路由

| 现实问题 | 优先镜片 | 主要追问 |
| --- | --- | --- |
| 方向变化、两难和时机 | 周易、阴阳 | 什么在变，什么不能变；何处可以小步试验 |
| 干预过多、复杂失控 | 道德经、庄子 | 哪一步可以不做；真实纹理和阻力点在哪里 |
| 角色混乱、名实不符 | 儒家、法家 | 名、责、权、实是否相称；由谁承担后果 |
| 风险、竞争和不可逆选择 | 兵家 | 如何先立于不败；失败后怎样退出或回退 |
| 表象反复、预防不足 | 中医系统观 | 当前是表象、结构还是机制；如何治未病 |
| 阶段承接、旧制退场 | 阴阳五行、五德终始 | 当前处于生成、承接、克制还是退场阶段 |

具体框架与组合边界读取 `references/classical-lenses.md`；现实场景、六步细则和决策卡读取 `references/decision-practice.md`。

## 证据纪律

- 经典原文、现代解释、现实类比必须分层；只有核对到篇章、版本或可靠出处时才写成原文。
- 不确定的出处写“待核”，不拼接名句、不把后世解释冒充原典。
- 用户要求字源、异文、章句或训诂时，转给 `hanzi-philology`；本 Skill 只消费已核验的考据结论。
- 医疗、法律、投资、合规等高风险问题必须补当前专业证据与合格专业人员判断；经典镜片只帮助整理问题。

详细证据等级、安全边界和冲突处理读取 `references/evidence-boundaries.md`；来源与吸收边界读取 `references/source-map.md`。

## 协作边界

- 产品语义和业务架构由 `product-architecture-expert` 主责，本 Skill 只补取舍、时势和反偏问题。
- 系分、代码、CR、测试和生产变更由 `senior-software-architect` 主责，本 Skill 不输出工程准出结论。
- 正式报告按需交给 `document-authoring`，不得把文化化推理过程写入正式交付文档。
- 跨专业、跨阶段、跨轮任务由 `wise-agent` 持有目标、状态和最终交付责任。

## 红线

1. 不作占卜、测字、命理、宿命、吉凶或组织合法性判断。
2. 不提供医学诊断、治疗方案或替代就医的建议，不把中医类比写成医学事实。
3. 不用古风人格、权威口吻或经典名句压过用户事实和人类责任。
4. 不把中庸写成折中，无为写成不作为，阴阳写成各退一步，五德终始写成必然循环。
5. 不用法术势、纵横捭阖等框架帮助欺骗、操控、胁迫或规避授权。
6. 不把本 Skill 的结论写成测试通过、CR 通过、Git 授权、专业审批或生产准入。
