# 产品架构专家技能树

本技能树用于能力评估、学习路径、方案审查和任务拆解。遇到具体任务时，只加载与任务相关的分支；不要把技能树整段复述给用户。支付与资金系统是重点垂直分支之一。

## 使用时机

- 用户要求评估产品专家能力、拆解学习路径、做岗位能力画像、规划产品架构能力地图，或需要判断某个复杂任务应落到哪个产品能力分支。

## 不适用场景

- 用户要求直接产出 PRD、支付资金方案、图形化交付或专项评审时，优先读取对应执行 reference，不要先复述技能树。

## 读取后必须产出

- 相关能力分支、当前成熟度判断、短板、下一步训练或交付任务，以及需要继续读取的 reference。

## 需要继续读取的 reference

- 通用产品架构读 `product-architecture-methodology.md`；PRD 读 `product-design-and-prd.md` 和 `product-prd-template.md`；支付资金读 `payment-scenario-routing.md` 和 `payment-methodology.md`；图形化读 `diagram-output.md`。

## 按任务读取索引

| 任务 | 优先读取 | 跳过 |
| --- | --- | --- |
| 产品架构能力评估 | 产品架构基础能力、复杂业务建模能力、产品治理能力、能力分级 | 支付垂直深项，除非命中资金场景 |
| 产品经理方法论 / 基础能力补齐 | 产品经理基础方法能力、产品方案与 PRD 能力、能力分级 | 不把基础岗位清单当作专家能力终点 |
| PRD/产品方案能力 | 产品方案与 PRD 能力、能力分级 | 支付与资金分支 |
| AI 产品工作成熟度评估 | AI 产品工作流成熟度、产品治理能力、能力分级 | 不把工具熟练度当成熟度 |
| 支付资金能力评估 | 支付与资金产品架构能力、能力分级 | 可扩展垂直领域 |

| 任务拆解 | 匹配当前任务的能力分支，并回到对应 reference | 全量技能树复述 |

## 1. 产品架构基础能力

- 产品定位：目标用户、业务目标、价值主张、范围边界、非目标和成功指标。
- 场景分析：用户场景、业务场景、运营场景、异常场景、跨系统协作场景。
- 能力地图：前台能力、后台能力、数据能力、风控能力、集成能力和运营能力。
- 业务对象：对象定义、字段口径、生命周期、状态、关系、权限和审计。
- 业务流程：主流程、逆向流程、异常流程、人工流程、跨系统流程和通知流程。
- 业务规则：权限、额度、计费、审批、通知、风控、数据留存、报表口径和规则版本。
- 验收设计：用户验收、业务验收、运营验收、数据验收、风险验收和技术交接条件。

## 1A. 产品经理基础方法能力

- 文档分型：能区分 BRD、MRD、PRD、产品方案、评审材料、上线复盘的目标读者、决策用途、详细程度和版本责任。
- 流程表达：能按目标选择一般流程图、泳道图、状态机、用户旅程、服务蓝图或产品架构图，并标注角色、触发、分支、异常、人工兜底和 SLA。
- 原型注释：能在原型中说明页面结构、输入输出、状态、校验、权限、异常、埋点、待确认项和与 PRD 的追踪关系。
- 用户研究：能把问卷、访谈、画像、反馈和工单转成问题地图、证据强度、场景频率、影响范围和验证假设。
- 需求管理：能维护需求池、父子需求、价值评估、优先级、评审会议、研发过程跟踪、上线复盘和需求关闭标准。
- 数据分析：能用指标维度、漏斗、增长模型、埋点和报表支撑目标设定、验收、运营监控和复盘。
- 技术/项目协作：能把产品语义交接给技术和项目团队，覆盖对象、流程、规则、数据、验收、风险 owner、依赖和发布节奏。
- 行业商业分析：能把行业、商业模式、竞品和公司背景拆成目标、主体、价值链、收入成本、风险和可验证机会。
- 知识库与能力复制：能把方法、模板、常见问题、案例经验和 source-map 沉淀为可复用参考，而不是停留在个人经验。

## 2. 产品方案与 PRD 能力

- 产品方案：背景目标、范围边界、用户角色、能力地图、对象模型、流程、规则、风险和验收。
- PRD：用户故事、功能说明、页面/交互说明、异常处理、权限、埋点、报表和验收标准。
- 文档治理：区分 PRD/MRD/BRD、目标读者、主文档/附录裁剪、版本记录、评审闭环和文档-原型-验收-工程交接同步。
- 功能分配：把功能识别为对象对外可见、有价值的交互行为，追溯到目标、父能力或上层业务对象，再分配到对象、流程、规则、数据和验收。
- 状态机：状态定义、进入条件、退出条件、可执行动作、终态、审计和异常迁移。
- 规则矩阵：规则对象、触发条件、判断逻辑、动作、优先级、版本、例外和验收样例。
- 运营后台：查询、审核、复核、补录、关闭、导出、权限、脱敏、日志、告警和工单。
- 数据产品：指标口径、事件埋点、报表模型、导出权限、数据质量和追溯。
- 评审材料：产品评审、技术交接、运营评审、风险评审、上线验收和复盘材料。
- 图形化表达：按产品目标选择能力地图、用户旅程、服务蓝图、流程图、状态机、资金四流和指标链路；AI 辅助画图只作为可编辑草案，必须回到 PRD、规则、证据和验收。

## 3. 复杂业务建模能力

- 多角色协作：用户、客户、商户、平台、运营、客服、风控、财务、法务、合规和外部机构。
- 多状态流转：创建、待处理、处理中、成功、失败、取消、关闭、冻结、解冻、归档。
- 多规则版本：规则配置、审批、灰度、回滚、历史版本、命中记录和审计追踪。
- 多系统集成：API、消息、文件、回调、批处理、数据同步和人工兜底。
- 多口径数据：业务口径、财务口径、运营口径、风控口径、监管口径和报表口径。
- 多风险边界：资金、合规、隐私、安全、权限、生产运营、客户体验和外部合同。

## 4. 产品治理能力

- 权限治理：角色、资源、动作、数据范围、职责分离和四眼复核。
- 审计治理：操作日志、前后值、规则版本、数据来源、访问审计和证据导出。
- 风险治理：风控规则、人工审核、误伤处理、告警升级、损失复盘和运营看板。
- 兼容治理：字段演进、接口兼容、历史数据、灰度、回滚和用户告知。
- 数据治理：指标字典、数据血缘、报表口径、数据质量、脱敏和留存策略。
- 上线治理：验收矩阵、依赖清单、SOP、培训、监控、应急预案和复盘机制。
- AI 产品工作流成熟度：区分 AI 工具提效和 AI-shaped 工作重构，能评估业务优势、流程变化、上下文结构化、任务编排、人工责任、验证周期、决策质量和返工率。

## 5. 支付与资金产品架构能力

支付与资金是本技能的高成熟度垂直分支。进入该分支前应先读取 `payment-scenario-routing.md`，涉及真实资金时先读取 `regulatory-baseline.md`。

### 5.1 支付基础认知

- 支付四流：业务流、支付信息流、账户/账务流、真实资金流。
- 支付五层：支付渠道层、支付网关层、支付核心层、统一支付能力层、支付接入层。
- 跨境支付五层：交易层、支付处理层、代理结算层、清算网络层、最终清算层。
- 支付十二字能力地图：买、收、付、退、充、提、转、调、算、结、管、对。
- 支付账本观：支付本质是多方账本、账户归属和真实资金路径的协同变化，信息流成功、账务入账、真实资金划拨和最终结算要分开表达。
- 支付角色：付款人、收款人、商户、平台、支付机构、银行、清算机构、通道、服务商、分账方、财务、监管。
- 支付方式：银行卡、快捷、网银、扫码、钱包余额、代扣、代付、转账、预授权、聚合支付、本地支付方式、VCC/虚拟卡。
- 支付单据：业务单、支付单、退款单、通道单、内部清算单、结算单、出款单、对账差错单、调账单、会计凭证。
- 交易异常：撤销、退款、冲正、补单、退汇、拒付、争议、重复扣款、超时、单边账。
- 金额口径：订单金额、优惠金额、实付金额、手续费、税费、分账金额、可结算金额、冻结金额、退款金额、损益金额。

### 5.2 支付产品能力

- 收款产品：收单、聚合支付、支付路由、支付确认、通知补偿、支付成功率优化、通道成本优化。
- 全球支付编排：把卡轨、钱包、本地支付、APM/RTP/A2A、PSP/网关、认证、风控、结算、争议和资金控制组织成可运营网络；竞争力看全球覆盖、本地适配、数据与风控、资金控制和争议治理的组合。
- 付款产品：代付、提现、批量付款、企业付款、工资/佣金/供应商付款、失败退回。
- 退款产品：原路退款、部分退款、跨周期退款、结算后退款、退款资金来源和退款失败处理。
- 争议产品：退款/争议碰撞、拒付、证据提交、交易争议举证行为日志、EvidencePackage、representment、商户准备金、负余额追偿。
- 分账分润：实时分账、延迟分账、比例/固定金额、服务商分润、平台佣金、补贴和税费处理。
- 商户产品：入网、资质审核、费率配置、结算配置、风险等级、资料变更、门店/终端管理。
- 卡与 VCC 产品：企业卡、员工卡、共享额度卡、一次性虚拟卡、旅行卡、供应商付款卡、spend controls。
- 卡组织清结算：四方/三方模式、BIN/IIN 路由、授权前置裁决、Authorization Core、Stand-In/SAF、open-to-buy、Financial Presentment、Matching/ARN、费用拆分、Posting、生命周期对账、reference continuity、replay / investigation、结算与网络费用。
- 外卡收单风控：商户准入、3DS/规则/评分、capture / fulfillment 控制、保证金/延迟结算、争议反馈和商户风险闭环。
- 三方支付机构：接入层、业务层、交易层、支付处理层、风控层、通道/渠道层，覆盖收付退清结账。
- 运营后台：交易查询、出款审核、冻结解冻、调账审批、对账差错处理、报表导出、操作审计。

### 5.3 账户、账务、清结算与对账

- 账户模型：客户账户、商户账户、平台内部账户、待结算账户、在途账户、差错账户、存管/客户备付金账户。
- 余额模型：可用、冻结、待结算、在途、已结算、挂账、长款、短款。
- 账务事件：入账、出账、冻结、解冻、扣减、冲正、结转、核销、调账。
- 账务矩阵：业务事件、触发条件、会计借方、会计贷方、余额变化、真实资金动作、金额口径、对账来源。
- 账务核心：外围驱动、凭证规则、账户结构、会计循环、总分核对和日切批处理。
- 清算/结算辨析：按理论、机构命名、平台产品和内部核算语境定义清算、结算、清结算。
- 支付清算生态：交易平台层、支付服务层、清算服务层、金融服务层、央行/金融基础设施层。
- 清结算全局规划：从多业务线资金乱象、线下核算、报表不记账和对账无闭环出发，按清算、账务、账户、结算、对账五中心切能力边界，并设计分期迁移路线。
- 跨机构清算：联机交易、实时清算、定时结算、日终处理。
- 对账能力：交易对账、账务一致性核对、通道对账、银行/存管资金对账、财务总分核对。
- 清结算能力：清分规则、结算规则、出款规则、结算批次、审核、重跑、差错核销和报表口径。

### 5.4 风控、监管与新支付形态

- 牌照边界：支付许可、清算业务、银行业务、预付卡、外汇/跨境、虚拟资产、本地法域许可。
- 客户资金保护：客户备付金、待结算资金、平台自有资金、保证金、存管账户、资金隔离。
- KYC/KYB/KYT/KYA：身份、商户/业务、交易、地址和持续监控，贯穿准入、交易、结算、运营和退出。
- AML 与反欺诈：名单筛查、可疑交易、交易限额、异常退款、频繁提现、账户接管、盗刷、credit-push fraud。
- 跨境与全球支付：Swift/GPI、代理行、本地清算网络、多币种账务、汇率、费用、到账时效和合规材料。
- 稳定币/Web3：链上交易、链下账务、托管钱包、出入金、Travel Rule、制裁筛查和地区限制。
- AI 代理支付：授权主体、授权范围、额度、次数、有效期、人工确认阈值、撤销机制和责任归属。

## 6. 可扩展垂直领域

后续可按同样方法扩展其他复杂业务领域：

- SaaS 订阅计费：套餐、权益、用量、账单、试用、续费、欠费、发票和收入确认。
- 交易市场：撮合、订单、履约、担保、评价、售后、纠纷和平台治理。
- 会员与权益：会员等级、权益包、发放、核销、过期、补偿和营销规则。
- 供应链与履约：采购、库存、仓配、结算、异常、协同和可追溯。
- 数据产品：指标体系、报表、权限、数据血缘、数据质量和经营分析。
- AI 产品工作流：任务编排、人工确认、权限边界、审计、评估和安全兜底。

## 能力分级

- **L1 入门**：能识别用户、主体、目标、流程和基础对象，输出简单 PRD。
- **L2 产品设计**：能独立设计能力地图、对象模型、流程、状态机、规则和验收标准。
- **L3 产品架构**：能处理多角色、多状态、多规则、多系统协作的复杂产品，并输出可交付方案。
- **L4 风险与治理**：能识别资金、合规、隐私、安全、运营和数据风险，并建立审计、报表、权限和上线治理机制。
- **L5 垂直领域专家**：能在支付与资金等高复杂领域设计跨业务线、跨主体、跨法域、跨系统的产品架构，并沉淀长期治理体系。
