# PRD 运营、通知、数据与发布附录

本文是 `product-prd-template.md` 的运营、通知、数据指标和发布计划附录。用于 PRD 涉及运营后台、人工处理、消息通知、埋点报表、发布灰度或运营准备时按需读取。

## 使用时机

- PRD 涉及运营后台、人工审核、复核、工单、告警、导出、审计、通知、消息、任务、数据指标、埋点、报表或发布运营计划。
- 标准 PRD 主骨架已经覆盖目标、角色、对象、流程、规则和验收，但需要补齐运营闭环和数据闭环。

## 不适用场景

- 轻量需求只需说明核心功能和验收，不涉及后台、报表、通知或发布节奏时，可不读取本文。
- 支付资金 PRD 的账务、对账和资金风险继续读取 `product-prd-financial-appendix.md` 和支付专项 reference。

## 读取后必须产出

- 运营后台、人工处理、通知消息、指标口径、埋点事件、报表导出、数据交付语义契约、发布阶段和运营准备项。

## 需要继续读取的 reference

- 可复制 PRD 主模板读 `product-prd-template.md`；质量门禁读 `product-prd-quality-gates.md`；支付资金专项读 `product-prd-financial-appendix.md`。

### 3.4 数据交付语义契约

数据需求交给数据开发、建模或 SQL 生成器前，先形成可评审的**数据交付语义契约**：

| 契约项 | 必须明确 |
| --- | --- |
| 目标与口径 | 业务目标、使用场景、指标定义、纳入与排除边界 |
| 统计结构 | 粒度、维度、时间窗口、时区、统计周期 |
| 来源映射证据 | 来源表/字段、关联关系及其元数据、Schema、血缘或已确认文档证据 |
| 处理规则 | 过滤条件、分区与新鲜度、去重与空值规则、退款/撤销等逆向口径 |
| 验收责任 | 样本或历史验收基线、数据 owner、业务 owner、已确认项和待确认项 |

- 来源、口径或规则未知时标为待确认并说明影响，**不得猜测表名、字段或业务规则**。
- 稳定业务指标口径进入项目受治理的数据字典、语义层或指标知识库；Skill reference 只保留跨项目可复用的方法、检查项和门禁。Schema、分区和血缘等实时元数据通过工具查询；模型取舍和场景规则由相应 owner 确认。
- 澄清、资源映射、模型决策和构造输入可按复杂度拆分或合并；只在业务语义和难逆模型决策处设置人工门禁，不机械复制固定阶段。
- 待确认项必须按影响分级：凡影响指标定义、粒度、来源映射、关联、过滤、时间窗口、统计周期或时区，均为**阻断性待确认项**，停止 SQL 或下游构造，待相应 owner 确认后恢复；不影响正确构造与验收的**非阻断待确认项**可作为显式风险随交接保留，不得伪装成已确认结论。
- 交给数据开发或 SQL 生成器的可执行输入只保留已确认语义，不携带讨论过程；非阻断待确认项单列在交接风险中。**生成 SQL 不是完成证据**，仍须核对 Schema、编译、血缘、样本基线、数据质量、性能成本和业务验收。
