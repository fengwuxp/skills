# 产品 PRD 文档模板

本文是产品架构专家的通用 PRD 模板，用于把业务需求整理成可评审、可开发、可测试、可运营的产品文档。主模板适用于多数 B 端、平台型、内部系统、数据产品、业务中台和复杂 C 端产品；支付与资金、金融、合规强约束场景使用第 20 章专项附录。

使用时可以按项目复杂度裁剪或合并章节，但不得缺失目标、范围、角色、对象、流程、规则、数据、风险和验收这些核心信息。

模板占位符使用 `〈...〉` 表示。实际输出时，以保障用户阅读体验为前提，正文段落和表格单元格默认保持普通文本，不对占位符、字段、状态、枚举、示例值、角色、能力项或风险项加粗，也不使用代码背景；只有接口路径、技术字段、配置键、协议字段、风险等级或需要强调的结论才例外。标题层级不受此规则影响，继续使用 Markdown 标题表达结构。

## 使用与裁剪指南

| 模式 | 适用场景 | 建议处理 |
| --- | --- | --- |
| 轻量型 | 小功能、低风险体验优化、内部工具的小改动 | 保留一页摘要、目标范围、场景、功能、验收；对象、流程、规则可合并描述。 |
| 标准型 | 多角色、多流程、多状态的常规业务需求 | 使用主模板第 0-19、21-22 章；第 20 章默认不使用。 |
| 强治理型 | 涉及权限、敏感数据、审批、运营后台、审计、生产风险的产品 | 保留权限、安全、审计、运营后台、风险依赖、发布运营和质量门禁。 |
| 金融资金型 | 涉及真实资金、账户、账本、清结算、对账、支付通道、银行卡、ACH、VCC、跨境或金融合规 | 必须使用第 20 章专项附录，并进入支付与资金专项审查。 |
| 数据/报表型 | 涉及指标体系、埋点、BI、导出、数据质量和口径治理 | 强化第 15 章，明确指标口径、数据源、权限、留存和质量校验。 |

## 1. 一页摘要

> 用 5-10 行说明本需求解决什么问题、服务哪些用户、交付哪些能力、成功标准是什么、主要风险在哪里。

- **问题**：〈当前痛点、机会或约束〉
- **目标用户/主体**：〈谁使用、谁受影响、谁负责验收〉
- **核心方案**：〈一句话描述产品方案〉
- **本期范围**：〈本期做什么〉
- **不做范围**：〈明确排除项〉
- **成功指标**：〈业务指标、用户指标、运营指标、质量指标〉
- **主要风险**：〈业务、技术、合规、运营、数据、依赖风险〉
- **上线策略**：〈灰度、白名单、全量、试点、仅内部使用〉

状态机红线：

- 不允许多个业务含义混在一个状态里。
- 不允许只有成功路径，没有失败、取消、超时和人工处理状态。
- 不允许状态变化缺少触发事件、操作者、时间和来源。
- 终态不得被普通操作随意改写。
