# 支付风控、欺诈与商户运营参考

本文用于设计支付产品中的交易风控、欺诈治理、商户风险运营和证据闭环。它不替代 AML、制裁筛查、消费者保护、卡组织规则、Nacha 规则或本地监管要求。

## 适用场景

- 设计收单、代扣、代付、钱包、发卡、企业卡、ACH/银行转账和本地支付方式的风险控制。
- 设计商户入网、KYB、风险评级、持续尽调、结算延迟、准备金和商户资料变更。
- 设计欺诈监测、交易拦截、人工审核、风险告警、风控规则治理和模型反馈闭环。
- 设计 ACH return、卡拒付、异常退款、重复出款、盗刷、账户接管和信用推送欺诈监测。

## 商户生命周期

- **准入前**：识别经营主体、受益所有人、MCC、经营网址/App、履约模式、地区、商品服务、禁限售行业和历史风险。
- **入网中**：完成 KYB、资料验证、合同签署、费率/限额/结算配置、风险等级、通道能力开通和责任边界确认。
- **运营中**：监控交易量、客诉、拒付率、退款率、ACH return 率、异常高峰、资料变更和经营范围变化。
- **风险处置**：支持结算暂停、延迟结算、保证金/准备金、限额、下架支付方式、人工审核、终止服务和资料补充。
- **退出后**：保留必要证据、处理未完结退款/拒付/争议、释放准备金、生成结算尾款和审计记录。

## 交易风险信号

- **用户信号**：账号年龄、身份验证强度、登录设备、IP、地理位置、历史交易、支付工具变化和异常行为。
- **商户信号**：MCC、行业、履约周期、客诉率、退款率、拒付率、异常增长、批量小额/大额交易和资料变更。
- **支付信号**：卡 BIN、卡品牌、卡类型、AVS/CVV/3DS 结果、ACH routing/account、钱包 token、支付方式风险等级。
- **行为信号**：速度、频次、金额跳变、同设备多账号、同卡多账号、同收款账户多主体和失败后重试模式。
- **资金信号**：退款异常、结算前高额退款、负余额、出款集中、收付款闭环异常、跨境路径异常和可疑资金回流。

## 风控决策层级

- **交易前**：商户准入、支付方式可用性、额度、行业限制、地区限制、名单筛查和强认证策略。
- **授权/扣款时**：实时评分、规则命中、3DS/强认证、人工审核、拒绝、限额、冻结或放行。
- **交易后**：延迟结算、准备金、异常退款拦截、可疑交易调查、争议证据补齐和模型反馈。
- **结算前**：按商户、通道、币种、批次、争议订单、负余额和资金风险决定是否可结算。
- **运营后**：复盘拒付、欺诈、投诉、差错和损失，把结果反哺规则、模型、准入和通道路由。

## ACH 与银行转账风险补充

- debit/代扣场景必须说明授权取得、授权留存、授权撤销和 unauthorized return 处理。
- credit-push 场景要关注社工欺诈、账户接管、收款人变更、供应商改账、异常大额转账和追回机制。
- return、NOC、reversal、refund、retry 的语义不同，不能共用一个失败处理动作。
- Nacha 规则会持续更新，涉及具体生效日期、监测义务和 return 代码时必须以最新 Nacha Operating Rules 为准。

## 规则与模型治理

- 规则要有名称、目的、字段、阈值、动作、版本、适用范围、审批人、上线时间和回滚方式。
- 模型结果要能解释输入信号、风险等级、决策动作和人工复核路径，不能成为不可审计的黑盒扣款/拒付依据。
- 风控动作要区分拒绝、暂停、人工审核、强认证、限额、冻结、延迟结算、准备金和关闭商户。
- 规则调整前后应监控通过率、转化率、欺诈率、拒付率、误杀率、人工审核量和损失金额。

## 运营与证据闭环

- 建立 case management，记录告警、证据、处理动作、责任人、时限、沟通记录和最终结论。
- 风控证据至少关联交易、用户、商户、支付工具、通道事件、账务事件、退款/争议和客服工单。
- 人工审核要有 SLA、升级路径、四眼复核和敏感操作留痕。
- 风险看板至少覆盖交易量、通过率、拒绝原因、欺诈损失、拒付率、退款率、ACH return 率、商户风险分布和准备金余额。

## 设计落地检查

1. 是否定义商户准入、持续尽调、资料变更、风险等级和退出机制。
2. 是否覆盖用户、商户、支付工具、设备、地理、金额、频次、退款和出款等风险信号。
3. 是否区分交易前、交易中、交易后、结算前和运营后的风控动作。
4. 是否说明规则/模型版本、审批、灰度、回滚、误杀复盘和指标监控。
5. 是否覆盖 ACH return、卡拒付、异常退款、账户接管、重复出款和 credit-push fraud。
6. 是否把风控证据与争议、对账、结算、客服和审计串起来。
7. 是否把 AML、制裁筛查、卡组织/Nacha/通道协议和本地监管列为待确认项。
