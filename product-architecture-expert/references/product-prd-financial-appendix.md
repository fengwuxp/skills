# PRD 支付与资金专项附录

本文是 `product-prd-template.md` 的支付与资金专项附录。仅在 PRD 涉及真实资金、账户、账本、清结算、对账、商户结算、分账分润、银行卡/卡组织、ACH/银行转账、VCC、风控欺诈、争议拒付、跨境支付、稳定币/Web3 或 AI 代理支付时读取。

## 使用时机

- 生成、补全或评审涉及支付、资金、金融合规、账户/账本、清结算、对账、卡组织、ACH、VCC 或跨境的 PRD。
- 通用 PRD 已覆盖目标、角色、对象、流程、规则、数据和验收，但缺少金融业务主轴、四流、账务矩阵或资金风险红线。

## 不适用场景

- 不涉及真实资金、金融账户、外部支付轨道或合规强约束的普通业务 PRD。
- 需要完整支付产品方法论时，继续读取 `payment-methodology.md`、`clearing-settlement.md` 或 `payment-design-checklists.md`。

## 读取后必须产出

- 主体、法域、资质、资金归属、四流、账务矩阵、对账依据、外部规则核验状态和金融风险待确认项。

## 需要继续读取的 reference

- 支付场景路由读 `payment-scenario-routing.md`，监管与外部规则读 `regulatory-baseline.md`，账务/清结算读 `clearing-settlement.md`。

## 4. 资金与风险红线

- 不混同客户资金、商户资金和平台自有资金。
- 不绕过 KYC/KYB/AML、名单筛查、交易限额、商户风控和可疑交易监测。
- 不把授权成功、清算成功、结算完成、资金可用和商户出款混为一个状态。
- 不对卡组织、ACH、本地清算网络、银行/通道协议、PCI DSS、税务或会计规则给出未经核验的确定结论。
- 争议、拒付、退款、冲正、return、reversal 和调账必须有证据、状态、责任方和账务影响。
