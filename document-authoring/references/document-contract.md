# 文档交付契约

## 使用时机

创建、重写、合并或最终收口正式文档前读取。信息不足时只补一个当前主 blocker，不一次索要全部细节。

## 最小契约

```text
文档类型：报告 / 制度 / 手册 / 研究说明 / 总结 / 其他
目标读者：谁阅读、谁评审、谁使用
目的：阅读后要理解、判断或执行什么
范围：包含什么；明确不包含什么
事实源：文件、数据、会议记录、链接、领域证据卡
文档状态：草案 / 评审版 / 最终版
输出载体：Markdown / DOCX / PDF / PPTX / 表格
Owner：内容确认、专业确认、最终批准分别由谁负责
验收：结构、事实、引用、修订、渲染和签批要求
```

## 文档类型最小骨架

| 类型 | 必需内容 |
| --- | --- |
| 报告 | 背景与目标、读者、来源、事实、分析、结论、行动、风险、验证 |
| 制度 | 目的、范围、定义、角色、规则、流程、例外、记录、复审 |
| 手册 | 读者、前置条件、步骤、预期结果、验证、故障处理、限制 |
| 研究说明 | 问题、范围、来源、事实、推断、不同观点、结论、局限 |

未知内容保留“待确认 + owner + 影响”，不得用常见做法或模型记忆补成事实。
