# 格式与渲染

## 使用时机

所有正式文档均读取；文档包含复杂表格、图片、分页、目录或页眉页脚时重点执行载体检查。

| 载体 | 默认处理 |
| --- | --- |
| Markdown | 作为可审查事实源，检查标题层级、表格、链接和代码块 |
| DOCX | 调用环境可用的文档能力，保留样式、表格、批注或修订要求，并渲染逐页检查 |
| PDF | 从稳定源生成或编辑，检查页数、字体、裁切、链接、图片和文本可提取性 |
| PPTX | 只有目标是演示或汇报时使用，检查每页信息密度、对齐和投屏可读性 |
| 表格 | 结构化数据交给表格能力，不把复杂数据硬塞进 Word 表格 |

## 简洁排版约规

- 对正文、标题、表格、文本框、形状或其它版面元素不添加背景色、底纹、色块、渐变或荧光笔效果。图片自身背景不属于新增格式，但不得另加装饰底板。
- 强调默认只用加粗，仅用于重要描述、关键结论或概念名词；标题按统一样式呈现，不整段连续加粗，不用文字颜色、下划线或斜体制造第二套强调层级。
- 表头不加底色，不使用斑马纹；仅用统一字重、细边框、对齐和留白区分表头与内容。
- 依靠标题层级、留白、短段落和对齐组织信息；同一语义层级保持相同字体、字号、字重和间距。
- 不使用装饰性文本框、卡片或横幅；只有正文无法清楚表达且载体确有需要时才使用图表或图片。

Markdown、HTML、文本或 DOCX 产物先运行 `scripts/check_document_style.py --file <path>`，检查显式背景色、底纹、高亮和文字颜色；该脚本不判断加粗内容是否语义重要，也不检查 PDF 像素。派生载体还必须检查：文件可打开、页数合理、目录和链接有效、表格不溢出、图片清晰、文字不截断、中文字体正常。逐页检查必须基于实际渲染页图或 PDF，记录文件路径、页数和异常页；页图出现方框、缺字或字体替换时，即使 PDF 文本可提取也判定失败，改用能访问所需字体的渲染器或回退 Markdown。工具不可用、未实际生成或无法渲染时，不声称视觉验收完成，交付 Markdown 并说明缺口。
