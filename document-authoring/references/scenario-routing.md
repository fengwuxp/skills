# 文档场景路由

## 使用时机

在开始正式文档任务前读取，用于判断本技能是否为正文 owner、只负责载体，还是应停止并路由专项 Skill。

| 用户任务 | 主责 | 本技能职责 | 停止条件 |
| --- | --- | --- | --- |
| 报告、制度、手册、研究说明、总结 | `document-authoring` | 从材料到正式文档闭环 | 目标读者、来源或验收方缺失 |
| PRD、产品方案、需求说明 | `产品架构专家` | 可在产品结论稳定后做载体和表达检查 | 产品对象、规则或验收未确认 |
| 系分、ADR、技术方案 | `资深架构师` | 可在工程结论稳定后做载体和表达检查 | 接口、数据、风险或验证未确认 |
| 字源、训诂、古文字研究 | `hanzi-philology` | 消费训诂证据卡并形成正式文档 | 来源、时代或争议状态不清 |
| DOCX/PDF/PPTX/表格编辑 | 环境可用的文档工具 | 提供内容契约和最终语义检查 | 工具不可用或无法渲染验证 |
| 简单润色、翻译、格式微调 | 当前 Agent | 直接完成最小修改 | 任务扩大为完整正式文档 |

复合任务先由领域 Skill 产出稳定事实，再由本技能在同一权威版本上组织正文，最后由载体工具生成和渲染文件。载体转换不得改变领域结论；PRD、系分等领域文档编辑完成后重新运行领域 Skill 的交付物检查，只对实际派生载体执行渲染检查，不套用本技能的报告、制度、手册或研究说明结构检查器。
